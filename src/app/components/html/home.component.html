About this application:

Welcome! This apllication allows you to keep records about cable components(wire coil, connector), created cables, prices, buys and orders. This section contains information on how to use the application, its possibilities and some others useful things. This application - seller side for 'Staff Buying Client' application.

1. Click on Wires Manager to create cable with an existing coil and connectors

  • For now,'type of signal' validation and 'number of wire cores equal number of connectors' validation are not implemented, so, please, be careful when choose components

  • The wire manager section contain two parts: cable production and cable storage(with possibility to edit/delete an existing cable)

  • You must understand, that the created cable is just potential cable(not physically existing), so you couldn't set wire count

  • You can doing primitive sorting by clicking on the table headers (name,first connector, length, second connector)

2. Click on Coils Manager to create coil that you could use in creating a cable

  • This section alomst like others sections. First part for creating coils, second part for inspect them, edit and delete

  • In this context, coils doesn't mean a big big real coils. It can be simple piece of wire

  • The coils as opossed to wires(in Wires manager) its a real coils

3. Click on Connectors Manager to create connector that you could use in creating cable

  • This section almost like others sections. First part for creating connectors, second part for inspect them, edit and delete

  • In this context, connectors are real connectors, so you could set the their count.

4. Click on Prices Manager to create prices for various cable components

  • This section alomst like others sections. First part for creating prices, second part for inspect them, edit and delete

  • Each new price associated with its own cable component and may have the same value with other prices

  • The price value must be set for one piece

5. Click on Buys Manager to create a buy

  • This is the most important section. The buy is a physically existing cables, so you won't be able create a buy if some of the components don't exist in the storage

  • The Buys are created, based on created wires. The prices for cable components are prices you set in Prices Manager

  • When the buy created, counts of all components that the wire contains is changing

  • You can view available components for the current cable, and the full price

  • You can add an image of the created wire which will be visible to the customer in the application for purchase cables(Staff Buying Client)

  • If you want to create custom buy(custom wire from the app - Staff Buying Client), you must click on 'isCustom' checkbox. Otherwise,the created buy will be displayed in the app for the customer

  • The description field is the information about the wire, that could be seen by customer in "Staff Buying Client" app. It may be how to use, where to use, inforamtion about manufacturer

  • By clicking on the table rows you can edit the current buy. So use it, when you have phiysically created an wire(change quantity), or you want to change an image, or description

6. Click on Orders Manager to edit existing orders

  • In the table of this section you will see any order created by customer in "Staff Buying Client" app.

  • Click on order you want in the table rows and you will see its details: Client's name and email, list of buys with needed and available count, order status selector and comment area

  • You can't create an order, change the list of buys, buys count or anything like that. You can only change the order status or delete an order

  • You can't select 'agree' status until the all 'available count' checkboxes is in 'check' value

  • You can't select the 'done' status if the current status is not 'agree'

  • You can't select the 'under consideration' status if the current status is not 'agree'

  • You can't select the 'canceled' status if the current status is 'done'

  • You can select the 'canceled' status only if you wrote the reason of cancel in the comment area

  • If the current status is 'under consideration/agree' and you changed it on 'agree/under consideration' - count of all buys contained in the order will change

  • If you see -999 in 'available count' column, it means, it is a custom buy, so, you must create that wire(and a buy with 'isCustom' check active) before you can click on checkbox

  • If the mail sending opportunity is setup, every time, when the order status changhed,the client receives a letter with the status and comments

For now, this application in developing stage. So, many features not implemented yet=((
